Tournament Rules

Setting up

  • Download portable OBS.
  • Unzip the file and run the RED RUNNER OBS LAUNCH.bat file to open OBS. This will contain the layout required if your race is being restreamed.
  • The restreamer will provide you with the server URL to stream to. Unless stated otherwise, you will NOT be streaming on your own channel for the race.
  • Join the Tournament Waiting Room voice channel 15-20 minutes before your race to set up your stream. This process is mandatory unless stated otherwise.
  • All racers must join the RaceTime channel created by the admins or commentators for race tracking and communication.
  • Streams must be live at least 5 minutes before the race starts.
  • Pay attention to the RaceTime.gg race window, especially before the race starts and during the first few minutes, as it is the primary form of communication between you and the restreamer/commentators.
  • Commentators/restreamers will communicate with you to set an initial game palette before the race begins. This is a countermeasure against pre-recorded runs. They should specify this in the form of a direction + optional button, e.g., LEFT+A or UP. Make sure you set this palette and are using it when you begin the race.
  • You may also be asked to do other actions at the discretion of the commentators if there is suspicion that your run may be pre-recorded.

Post-Race

  • After you finish your race, if you would like to join the stream for a post-race interview, please join the Tournament Waiting Room voice channel. Someone will then pull you into the restream/commentary channel.

Gameplay

  • The run has to follow the Game and Category rules from Speedrun.com.
  • Instant Text (IT) is banned from the tournament. In the event that you mistakenly get IT, it has to be killed off before the first turn on Bridge Rival and before the first turn on Dig Rocket respectively. Otherwise, you will be issued a 3 minute penalty.
  • Runs longer than 2h30m count as "Did Not Finish" (DNF) to keep on track with restreaming time slots.
  • In the unlikely event of a tie that ends up mattering for advancement or seeding, each run in question will be frame counted.

Crashes/Disconnects

  • In the event that a racer’s stream disconnects for an extended period of time, they will have to provide a VOD of the run to the admins for review.
  • If a racer’s console/emulator crashes, they must continue from their last save.
  • If a racer’s console/emulator crashes within the first 20 minutes, a rematch is allowed ONLY if all racers agree.
